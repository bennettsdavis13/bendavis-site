Your Website
Owner's Guide

Everything you need to fully own and run your site yourself, for free, forever. No coding. Take it one part at a time, about 30–45 minutes total. After that, you just use the editor.

How it all fits together

Your website lives on Vercel (free hosting, never expires). Your text & photos live in a GitHub filing cabinet. You edit everything from one hidden page on your site called the Editor, protected by a password only you know. Your domain is at Namecheap, pointed at Vercel. Change something in the Editor, hit Save, and your live site refreshes in about a minute.

2

Create your editor "key" (GitHub token)

This token is the key that lets your Editor save changes. You make it once.

  1. Go to Fine-grained tokens → Generate new token.
  2. Name: website-editor
  3. Expiration: choose No expiration (or the longest option, and set a reminder to redo it later).
  4. Repository access: Only select repositories → choose bendavis-site.
  5. Permissions: open Repository permissions, find Contents, set it to Read and write. Leave everything else.
  6. Click Generate token and copy it now (starts with github_pat_…). You only see it once — paste it somewhere safe for a few minutes.
Keep it private
This token is like a password. Never post it anywhere public. It only ever gets pasted into Vercel (next steps).
3

Sign up for Vercel & launch the site

  1. Go to vercel.com/signup → Continue with GitHub and authorize.
  2. Choose the Hobby plan — it's free and all you'll ever need.
  3. Dashboard → Add New… → Project.
  4. Find bendavis-site in your repo list → Import.
  5. Vercel auto-detects Next.js. Don't change any build settings.
  6. Open Environment Variables and add the five from Part 4 (do it now so it works first try).
  7. Click Deploy, wait about a minute, and you'll get a live link.
4

Set your password & keys (Environment Variables)

This is where your password lives — never in the code, just a private setting only you can see. To change it later, you edit ADMIN_PASSWORD here. Go to your project → Settings → Environment Variables and add these five (type the Name exactly, paste the Value, leave environment as "All", Save):

NameValueWhat it is
ADMIN_PASSWORDyou chooseYour Editor login password. Make it strong.
SESSION_SECRETrandom text30+ random characters (keyboard mash). Keeps logins secure.
GITHUB_TOKENfrom Part 2Your github_pat_… token. Lets the Editor save.
GITHUB_OWNERyour usernamee.g. bennettsdavis.
GITHUB_REPObendavis-siteYour repository name.

Then go to the Deployments tab → … on the latest deployment → Redeploy. (You only redeploy when you change these settings, not for normal edits.)

Test it
Visit your-site.vercel.app/admin, type your ADMIN_PASSWORD, and you should land in the Editor. If it says "not connected," re-check the variable names are spelled exactly and that you redeployed.
5

Connect your domain (Namecheap → Vercel)

  1. Vercel → project → Settings → Domains → type bennettsdavis.site → Add.
  2. Vercel shows the exact DNS records to add. Usually:
    • A record — Host @ → Value 76.76.21.21
    • CNAME — Host www → Value cname.vercel-dns.com
    Use the exact values Vercel shows you on screen.
  3. New tab: Namecheap → Domain List → Manage (next to bennettsdavis.site) → Advanced DNS.
  4. Delete any parking/redirect records, then Add New Record for each one Vercel gave you.
  5. Save. Back in Vercel it will show Valid Configuration within minutes to an hour, with a free SSL lock added automatically.

That's the last setup step. From here on you only ever use the Editor.

6

How to edit your site (the fun part)

  1. Go to bennettsdavis.site/admin and log in with your password (you log in fresh each visit).
  2. Click Go to my site. In the bottom-right corner you'll now see an Edit site button.
  3. Click Edit site to turn on Edit Mode. The whole site becomes editable in place.

In Edit Mode, anything you can edit highlights in blue when you hover. Then:

  • Change words: click any text and type, right on the page.
  • Swap a photo: click the photo and pick a new one.
  • Reorder photos: drag them around.
  • Add new stuff: under each section is an Add a block bar with + Text, + Photo, + Video, and + Button. Add as many as you like.
  • Change a video: use + Video, or paste any YouTube link into the "Video link or ID" field that appears under a page's main video in Edit Mode.
  • Remove a block: click the small red x on it.
  • Save: hit Save in the bottom bar. Your live site updates within about a minute.
Quick edits (email & links)
Your email and link URLs are not visible text on the page, so they live on a simple form. In the Edit Mode bar click Quick edits (or go to bennettsdavis.site/admin/quick) to change your booking email, social links, support/tips links, promo links, and SEO, and to + Add promo link.
Good to know
The editor is hidden and blocked from Google; only your password gets in. To change your password later: Vercel → Settings → Environment Variables → edit ADMIN_PASSWORD → Save → Redeploy.
7

Deployments & rolling back

Every time you save a change, your site publishes a new deployment — a complete snapshot of the whole site at that moment. Vercel keeps every one of them. You can see the list at Vercel → your project → Deployments, each row labelled with what changed:

Undo anything, anytime
If an edit ever goes wrong, jump back to any earlier version in two clicks: find the version you want in the Deployments list, click the ... menu on its row, and choose Instant Rollback (or Promote to Production). Your live site switches to that exact snapshot immediately, and you can roll forward again the same way. Nothing is ever lost, so you genuinely cannot break your site.

Troubleshooting

  • "Not connected": a variable is missing/misspelled — re-check Part 4, then redeploy.
  • "Wrong password": you're not typing your ADMIN_PASSWORD value.
  • Saved but no change: wait 1–2 min and hard-refresh (Ctrl/Cmd+Shift+R).
  • Photo won't upload: keep images under 8MB.
  • Domain not live yet: DNS can take up to an hour; Vercel's Domains page shows what's pending.

Bonus: edit with AI

When you want a bigger change than Edit Mode covers — a whole new section, a new tab, or a fresh look — you can just ask Claude to do it. This site ships with a guide file (AGENTS.md) so Claude understands the whole project instantly.

Quick walkthrough with Claude

  1. Open Claude, connect your GitHub, and point it at the bendavis-site repository.
  2. Describe what you want in plain English, for example:
    • "Add a Testimonials section to the home page with three quote cards."
    • "Add a new Press tab to the menu."
    • "Give the whole site a dark theme."
  3. Claude reads the project — it follows the built-in AGENTS.md guide, so it already knows your design and where everything lives — makes the change, and proposes it as a pull request (an update for you to review).
  4. Approve it (Merge). Your site rebuilds and goes live in about a minute.
Zero risk
If you don't like what the AI did, just don't approve it. And if something is already live, roll it back in two clicks (see Deployments above). Experiment as much as you want.
